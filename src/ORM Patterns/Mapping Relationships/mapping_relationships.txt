Типы маппинга
    Single Table inheritance
        Описание
            Дынные родительского и дочерних классов лежат в одной таблице.
            Есть поле типа для различия строк конкретных классов.
            Поля не используемые в конкретном классе держатся пустыми.
        Применение
            Способ удобен если различия между классами не велики.

    Class Table Inheritance
        Описание
            Каждый класс (включая абстрактный) дерева наследования имеет свою таблицу.
            Дочерние таблицы имеют ссылку на родительскую.
        Применение
            Так как дочерние таблицы не дублируют родительские поля подход хорош для экономии места.
            Удобно менять общие поля, так как изменение родителя не требует изменений в таблицах-наследниках.

    Concrete Table inheritance
        Описание
            Каждый конкретный (не абстрактный) класс дерева наследования имеет свою таблицу.
            Общие поля (те что в супурклассе) дублируются в наследниках.
        Применение
            Способ удобен когда производительность важнее используемой памяти, так как позволяет избежать лишних джойнов.

Принцип выбора
    Классы имеют минимальные различия                   - Single Table inheritance
    Важно используемое место и гибкость в изменениях    - Class Table Inheritance
    Важна производительность                            - Concrete Table Inheritance
